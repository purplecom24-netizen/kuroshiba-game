# 要件定義書: 「出来高+陽線」ルール v2.0(v1 からの派生ルール)

事前登録日: 2026-07-02
以後、判定基準・ルール定義の変更は本書末尾の変更履歴に記録すること。無断変更は検証の無効化とみなす。

## 0. 位置づけ

- 本書は [docs/requirements_v1.md](requirements_v1.md) の**差分定義**である。本書に記載のない項目
  (ユニバース、エントリー・出口の執行規則、サイズとコスト、ベースライン、Gate 1/2 判定基準、
  成果物、技術仕様、データ検証、QA要件、スコープ外・禁止事項)はすべて v1 を継承する。
- v2 は v1 とは**別の新ルール**であり、v1 の検証結果を上書き・置換するものではない。
  v1 のフォワードテスト(開始 2026-07-02)は本書と無関係に継続する。

## 1. v1 からの変更点(3点のみ)

| 項目 | v1 | v2 |
|---|---|---|
| 出来高倍率しきい値 | 2.0 | **3.0** |
| 時間切れ決済 | 5営業日目の終値 | **10営業日目の終値** |
| レジームフィルタ | なし | **シグナル日の終値 > 200日単純移動平均** |

- 200日単純移動平均は終値ベース、シグナル日当日を含む直近200営業日で計算する。
  営業日データが200日に満たない銘柄・期間はシグナル無効(285A.T は上場約10ヶ月後まで対象外)。
- スリッページ 0.3%・コスト片道 0.1% は v1 と同一(執行前提であり、ルールの一部として変更しない)。

## 2. 選定根拠(事前に明文化)

1. **単調な用量反応**: v1 の探索的感度分析(output/sensitivity.md, 27組合せ)で、
   出来高倍率(1.5→2.0→3.0)と保有日数(3→5→10)の両軸に対して期待値Rがほぼ単調に改善した。
   孤立した最良セルの選択ではなく、2軸の勾配の延長として選定する。
2. **文献との整合**: 高出来高リターンプレミアム(Gervais, Kaniel & Mingelgrin 2001)は
   数週間スケールの保有で報告されており、保有10日への延長はこれと整合する。
   長期トレンドフィルタは Faber (2007) 等で広く検証されている。
3. **コスト構造**: 執行コストは1トレードあたり固定であり、保有期間の延長は
   1日あたりコスト負担を軽減する。
4. **レジーム依存**: v1 の年次別成績はプラスが2023・2024(上昇相場)に集中し、
   2021・2022(軟調)はマイナスだった。200日MAフィルタはこの観察と Faber (2007) に基づく。

## 3. 証拠力に関する明示的な警告(最終レポートに必ず明記)

- **v2 のバックテスト合格は v1 のときより証拠力が弱い。** 変更3点はいずれも
  同一データ(2021-07〜2026-07)の観察から導かれており、v2 のバックテストは
  確認的検証ではなく**半探索的検証**である。
- 特に懸念されるのは勝者の呪い(選択された組合せは実力より上振れして見える)。
  フォワードテストの成績はバックテストを下回ると事前に予想する。
- **真の確認はフォワードテスト(Phase 2)のみ**が与える。
- v1 と同じ勝者バイアス(ユニバース選定)も引き続き適用される。

## 4. トレード数に関する事前確認

- v1 感度分析で倍率3.0系のトレード数は約110件だった。MAフィルタの追加でさらに減少し、
  **Gate 1 の「トレード数 ≥ 100」を満たさない可能性がある**。
  その場合は事前登録どおり「検出力不足・判定不能」とし、合否を出さない。
  基準の事後緩和は行わない。

## 5. 実行手順

1. `python backtest.py --rule v2` — 成果物は `output_v2/` に出力される
2. Gate 1 合格の場合のみ: `python daily_scan.py --activate v2`
   (フォワード台帳は `forward/v2/` に分離され、v1 と並走する)
