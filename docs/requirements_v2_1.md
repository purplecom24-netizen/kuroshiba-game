# 要件定義書: 「出来高+陽線」ルール v2.1(v2 の検証期間拡張)

事前登録日: 2026-07-02
以後、判定基準・ルール定義の変更は本書末尾の変更履歴に記録すること。無断変更は検証の無効化とみなす。

## 0. 位置づけ

- **ルール定義は v2([docs/requirements_v2.md](requirements_v2.md))と完全同一**である。
  変更するのは検証テストの仕様のみ。本書に記載のない項目はすべて v2(therefore v1)を継承する。
- 本登録の動機: v2 のバックテスト(2021-07〜2026-07)はトレード数 65 件で
  事前登録基準「トレード数 ≥ 100」を満たさず**判定不能**となった(2026-07-02 実行)。
  成績探索ではなく**検出力の確保**のための期間拡張であり、ルールには一切触れない。

## 2. 擬似アウトオブサンプル検証(本登録の核心)

- v2 のパラメータ(出来高3.0倍・保有10日・200日MAフィルタ)は、**2021-07〜2026-07 の
  データの観察**(感度分析・年次別成績)から選ばれた。
- したがって **2011-07-01〜2021-06-30 の10年間は、パラメータ選択に一切使われていない
  擬似アウトオブサンプル(OOS)期間**である。この期間の成績は、v2 の成績が
  「実在するエッジ」か「探索した窓への過剰適合」かを識別する検出器として機能する。

### Gate 1 判定(v1 の4条件に第5条件を追加。実行前に固定)

1. トレード数 ≥ 100(全期間。未満は判定不能)
2. コスト控除後の期待値 > 0(R建て・全期間)
3. 最終資産がランダム分布(同一期間で再生成)の95パーセンタイル以上
4. 最大ドローダウン ≤ 25%
5. **擬似OOS期間(2011-07-01〜2021-06-30 のシグナル)の期待値R > 0。
   該当トレードが30件未満の場合は判定不能とする。**

- 5条件すべて合格 → v2.1 ルールのフォワードテスト(`daily_scan.py --activate v21`)へ。
- 第5条件のみ不合格の場合、それは「2021年以降だけ機能して見えた」ことを意味し、
  **過剰適合の強い証拠としてルールを棄却する**。全期間成績が良好でも救済しない。

## 3. 留意事項

- 期間拡張により勝者バイアスの性質が変わる: 主ユニバースは「2024〜26年の勝者」で
  選ばれており、その 2011〜2021 年の成績は選定理由と直接結びつかない(バイアス弱め)。
  ただし全銘柄が現在まで生存している生存バイアスは残る。
- 285A.T(2024年12月上場)は従来どおり上場以降のみ。200日MAフィルタにより
  実質的なシグナル対象は上場約10ヶ月後から。
- 15年分のデータでは分割・データ異常の検出頻度が上がる想定。データ検証(§7)と
  グリッチ自動修復は従来どおり適用し、停止した場合は data_quality.md を人間が確認する。
- 中立ユニバース(TOPIX Core30)の併記も従来どおり実施する。

## 4. 実行手順

1. `python backtest.py --rule v21` — 成果物は `output_v21/` に出力される
2. Gate 1(5条件)合格の場合のみ: `python daily_scan.py --activate v21`
