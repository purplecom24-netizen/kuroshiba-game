# 「出来高+陽線」ルール検証システム v1.0 — 使い方

事前登録仕様: [docs/requirements_v1.md](docs/requirements_v1.md)(2026-07-02 登録・変更禁止)

## セットアップと実行

```bash
pip install -r requirements.txt
python backtest.py
```

1コマンドで 取得 → データ検証 → 先読み検査 → バックテスト(4系列)→ ベースラインA/B → Gate 1 判定 → レポート が完走する。

| オプション | 意味 |
|---|---|
| `--refresh` | `data/` のCSVキャッシュを無視して再取得 |
| `--trials N` | モンテカルロ試行数(デフォルト 1000) |
| `--qa-truncate N` | 先読み検査で切り落とす末尾営業日数(デフォルト 30、`0` で省略) |
| `--skip-sensitivity` | 感度分析(27組合せ)を省略 |

### 成果物(`output/`)

- `data_quality.md` — データ検証(§7)。重大な異常があれば本体を実行せず停止する
- `trades.csv` — 全トレード台帳(§5 の列定義どおり・UTF-8 BOM 付きで Excel 可読)
- `summary.md` — 成績・ベースライン比較・**Gate 1 判定結果**・前提と既知の限界
- `equity_curve.png` — ルール資産曲線 vs TOPIX vs ランダム5〜95%帯
- `random_dist.png` — ランダム最終資産ヒストグラム+ルール位置
- `sensitivity.md` — 感度分析(探索的・Gate 判定に不使用)

## QAテスト(§8 受け入れ基準)

合成データのみ使用・ネットワーク不要:

```bash
python -m unittest discover -s tests -v
```

- 先読みバイアス検査(末尾切り落とし再実行・シグナル/トレード完全一致)
- 出口ロジック: ギャップダウン損切り(始値約定)/ 同日両到達(損切り優先)/ 時間切れ / ギャップアップ利確
- サイズ計算: MIN_STOP_DIST スキップ / 余力超過時の縮小 / 単元制約(切り捨て・スキップ)
- 再現性: 同一入力・同一シード(42)で完全一致
- ポートフォリオ規則: 最大3枠・出来高倍率順採用・重複建て禁止

実データに対する先読み検査は本体実行時に毎回自動で行われる(`--qa-truncate`)。

## 仕様書が明示していない点の実装上の解釈

いずれも `summary.md` にも明記される:

1. 期間末に未決済のポジションは期間末終値で強制決済(出口理由 `eod`)し、統計に含める。
2. Gate 1 判定は**主ユニバース・端株可(理論値モード)**の結果に対して行う。中立ユニバース・単元制約モードは頑健性チェックとして併記。
3. データ品質の停止基準: 欠損率 > 5% / 出来高0 > 10日 / |日次リターン| > 35%(±30%超はすべてリスト化)。分割日当日の調整後リターンが ±20% を超える場合は調整ミスの疑いとして停止。
4. ベースラインA(TOPIX バイ&ホールド)は仕様の文言どおり執行コストなし(寄付買い→終値評価)。
5. 中立ユニバース(TOPIX Core30)の構成銘柄は実装時点の近似リスト。定期入替に合わせて `backtest.py` の `NEUTRAL_UNIVERSE` のみ編集可(ルール本体は変更しない)。

## 実行環境に関する注意

- Yahoo Finance へのHTTPSアクセスが必要(このリポジトリを作成したリモート実行環境ではネットワークポリシーによりブロックされていたため、実データでの実行はローカル環境で行うこと)。
- 取得データは `data/*.csv` にキャッシュされる。同一キャッシュ+同一シードなら出力は完全に再現される。
- Phase 2(`daily_scan.py`)は **Gate 1 合格時のみ**実装する(§9・§10)。
